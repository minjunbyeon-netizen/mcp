====================================================================
MCP 블로그 자동화 시스템 — 치명적 약점 및 보강 제안
====================================================================
작성일: 2026-02-24
현재 시스템: 페르소나 기반 블로그 자동 생성 (CLI + Web Dashboard)
====================================================================

1. 🩸 치명적 약점: 이미지 생성/매칭 부재

블로그는 글 반, 사진 반입니다. 현재는 텍스트만 생성해주는데,
실무자는 결국 적절한 이미지를 찾느라 시간을 다 씁니다.

보강 제안:
- 보도자료 내 이미지 자동 추출 및 배치
- 또는 Unsplash/DALL-E 연동하여 본문 문맥에 맞는 이미지 자동 추천

--------------------------------------------------------------------

2. 📉 확장성 한계: JSON 파일 저장소

지금은 데이터가 적어서 괜찮지만, 글이 100개, 1,000개 쌓이면
시스템이 느려지고 파일 깨짐(Data Corruption) 위험이 큽니다.
검색/필터링도 어렵습니다.

보강 제안:
- SQLite라도 도입하여 DB화 필수. (마이그레이션이 시급함)

--------------------------------------------------------------------

3. 🤖 피드백 루프 단절 (가장 중요)

현재는 "생성 → Google Docs 내보내기"로 끝입니다. 실무자가
Docs에서 수정한 내용을 AI가 다시 배우지 못합니다. 즉,
"쓰면 쓸수록 똑똑해지는" 구조가 아닙니다.
매번 똑같은 실수를 반복할 겁니다.

보강 제안:
- Docs에서 최종 수정된 버전을 다시 시스템으로 가져와서(Import),
  "초안 vs 최종본" 차이를 학습하는 파인튜닝(Fine-tuning)
  데이터셋 축적 기능.

--------------------------------------------------------------------

4. ⏳ 단건 처리의 비효율 (Batch 기능 부재)

에이전시는 보통 월초에 한 달 치(4~8건) 기획을 한꺼번에 합니다.
지금처럼 하나씩 입력하고 기다리는 건 너무 느립니다.

보강 제안:
- 엑셀/CSV로 보도자료 10개를 한 번에 업로드
  → 일괄 생성(Batch Generation) 기능.

--------------------------------------------------------------------

5. 📅 일정 관리(캘린더) 부재

블로그는 '언제 발행하느냐'가 핵심입니다. 현재는 '생성'에만
집중되어 있고 '관리' 기능이 약합니다.

보강 제안:
- 대시보드 메인에 달력(Calendar) 뷰 추가
- "2월 15일: 남구청 안심귀가 서비스 (작성중)" 처럼
  발행 스케줄링 및 상태(초안/검수/발행완료) 관리 기능.
